# UDPリモコン モーター制御 (XIAO ESP32-S3 Sense)

XIAO ESP32-S3 Sense がカメラでラインを検出して自律走行する。あわせて Mac(Processing) から
UDP でモーター出力(%)を送るリモコン駆動にも切り替えられる（同じ駆動経路を共有）。
ライン検出の状況はシリアルモニタ(115200)で常時確認できる。

## 構成

```
[Mac / Processing]  --UDP("L,R" %)-->  [ESP32-S3]  --ESC信号-->  [左右モーター]
   キーボードで%入力                      WiFi(SoftAP)
   バーで可視化                          フェイルセーフ停止
```

- `esp32_motor_udp/`  … ESP32-S3 に書き込むファーム（Arduino IDE）
- `processing_remote/` … PC で動かすリモコン（Processing）
- `モーターテスト/`    … 元のモーター単体テスト（参考）

## セットアップ

### ESP32-S3 側

対象ボードは **Seeed Studio XIAO ESP32-S3 Sense**（ESP32-S3, 8MB PSRAM, OV2640）。
カメラピンはこのボード前提でファイル冒頭の `CAM_*` に定義済み。別ボードなら `CAM_*` を差し替える。

1. Arduino IDE のライブラリマネージャで **ESP32Servo** をインストール
   （カメラ用 `esp_camera` は ESP32 コアに同梱）
2. `esp32_motor_udp/esp32_motor_udp.ino` を開く
3. ファイル先頭の設定を確認
   - `MOTOR_L_PIN=1` (D0) / `MOTOR_R_PIN=2` (D1) … カメラ非使用GPIO。配線に合わせて変更可
   - `LINE_IS_DARK` … 明るい床に黒線=`true` / 暗い床に白線=`false`
   - `BASE_SPEED` / `KP` / `MAX_TURN` … 走行の基本速度と旋回の強さ（要現場調整）
   - `AP_SSID` / `AP_PASS` … 自前WiFiの名前/パスワード
4. ボードを `XIAO_ESP32S3`、**PSRAM を `OPI PSRAM`** にして書き込み
5. シリアルモニタ(115200)で `カメラ初期化OK` と `ESP32 IP : 192.168.4.1` を確認

### Mac / Processing 側

1. Mac の WiFi を ESP32 の SSID（既定 `LineTracer`）に接続
2. `processing_remote/processing_remote.pde` を Processing で開く
3. `ESP32_IP` が `192.168.4.1`、`ESP32_PORT` が `4210` であることを確認
4. 実行

## 動作モード（シリアルで切替）

ESP32 はシリアルモニタに 1 文字送ると動作モードが切り替わる。**起動時は IDLE**。

| 入力 | モード | 動作 |
|------|--------|------|
| `i` | IDLE（既定） | モーターは回さず、ライン検出ログだけ出す。**車輪を回さず検証**できる |
| `l` | LINE | カメラのライン検出で自律走行。ライン消失で停止。**基準速度は Processing の UDP 値で変更可**（下記） |
| `r` | REMOTE | Processing からの UDP リモコンで駆動 |
| `s` | 停止 | 即停止して IDLE へ |
| `c` | 較正 | 床と線の輝度を実測してしきい値を自動較正（NVS 保存・再起動後も保持） |
| `x` | 較正消去 | 較正データを消して適応しきい値に戻す |

どのモードでも検出処理は毎ループ走るので、ログだけは常に出る。

## ライントレースの確認（シリアルモニタ）

シリアルモニタ(115200) に 150ms ごとに検出状況が出る。これを見れば
カメラが線を捉えているか・どう曲がろうとしているかが分かる。

```
[LINE] det=Y mean=128 cx= 92/160 err= +12 px= 340 base= 30% -> L= +41% R= +19% | cal=DARK@96 mode=IDLE (出力のみ)
[LINE] det=N mean=131 px=   3 -- ライン消失 -- | cal=off mode=LINE -> 停止
```

`cal=` は較正状態。`cal=off` は未較正（適応しきい値）、`cal=DARK@96` は較正済みで
「暗い線・絶対しきい値96」を意味する（`BRT` なら明るい線）。

| 項目 | 意味 |
|------|------|
| `det` | ライン検出有無（Y/N） |
| `mean` | 注目帯(ROI)の平均輝度。露出が極端なら照明やしきい値を調整 |
| `cx` | 線の重心X(0〜画像幅)。中央が幅の半分 |
| `err` | 中央からのずれ。右に外れると `+`、左で `-` |
| `px` | 線とみなした画素数。`MIN_LINE_PIX` 未満で「消失」 |
| `base` | LINE モードの基準速度%。既定は `BASE_SPEED`、Processing から変更可 |
| `L/R` | P制御で算出した左右出力%。LINE モードのときだけ実際に駆動 |

**検証手順**: 起動（IDLE）→ 車体を手に持ってカメラを線に向ける → `det=Y` と
`cx`/`err` が線の位置に追従して動けば検出 OK → `L=`/`R=` が線を追う向きに変化すれば
制御 OK → 床に置いて `l` を送れば自律走行。追従が弱い/振れるなら `KP` を調整する。
照明や床色でうまく線を拾えないときは、`LINE_MARGIN` を手で触るより**較正（`c`）が確実**。

## キャリブレーション（`c` / `x`）

線の明暗としきい値を**実測で自動決定**する。固定値（`LINE_IS_DARK` / `LINE_MARGIN`）の
手調整が要らなくなり、結果は NVS に保存されて**再起動後も保持**される。追加ライブラリは不要
（`Preferences` は ESP32 コア同梱）。

**手順**（車体を手に持って操作）:

1. シリアルモニタで `c` を送る
2. `[1/2]` … カメラを**床だけ（線が写らない位置）**に向けて任意のキーを送る
3. `[2/2]` … カメラを**線の上**に向けて任意のキーを送る
4. `>> 較正完了: DARK-line floor=131 line=58 thresh=94 (NVS保存済)` が出れば成功

床と線の輝度から、線が暗い側か明るい側かを自動判定し、両者の中間を絶対しきい値にする。
較正後は検出ログが `cal=DARK@94` のように較正値を表示する。

- **コントラスト不足**（`差=… < 25`）と出たら保存されない。照明を明るくする／線の太い箇所で撮り直す
- 各ステップは **15 秒待っても入力が無ければ中止**（既存の較正は変更しない）
- `x` を送ると較正を消去し、適応しきい値（`mean ± LINE_MARGIN`）に戻る
- 較正は **GRAYSCALE の輝度ベース**。コース・照明が変わったら再較正する

## リモコン操作（REMOTE モード時 / Processing）

| 入力 | 動作 |
|------|------|
| `50` + Enter | 両輪 50% 前進 |
| `50,30` + Enter | 左50% 右30%（差動・カーブ） |
| `-40` + Enter | 両輪 40% 後退 |
| `s` / Space | 即停止 (0%) |
| ↑ / ↓ | 両輪を ±5% 微調整して即送信 |

値の範囲は **-100〜100 (%)**。画面に現在の出力%とバーが表示される。

## LINE モードの速度を Processing から変える

`l`（LINE）モード中に Processing から値を送ると、**その値が自律走行の基準速度になる**
（左右独立で来た場合は左右の平均を採用）。操舵（曲がり）はカメラの P 制御が担うので、
Processing 側は **速度ツマミ**として使える。

| Processing 入力 | LINE モードでの効果 |
|------|------|
| `40` + Enter | 基準速度 40%（直進時 40、旋回時はここから±） |
| `0` / `s` | 基準速度 0 = 実質停止（ラインは検出し続ける） |
| ↑ / ↓ | 基準速度を ±5% 調整 |

シリアルログの `base=` で現在の基準速度を確認できる。値はモードで意味が変わる点に注意：
**REMOTE では左右出力そのもの／LINE では基準速度**。起動時の既定は `BASE_SPEED`(=30)。

## 通信プロトコル

- トランスポート: UDP（既定ポート 4210）
- ペイロード: ASCII テキスト
  - `"L,R"` … 左右独立（例 `"50,30"`、-100〜100 の整数）
  - `"V"`  … 単一値で両輪同値（例 `"50"`）
- 受信側の解釈はモード依存：**REMOTE は左右出力／LINE は基準速度（左右平均）**
- Processing は直近の指令を **200ms間隔で再送**（キープアライブ）

## フェイルセーフ

REMOTE モードでは最後の有効指令から **500ms** 指令が途切れると自動で 0% 停止する
（`FAILSAFE_MS`）。通信断やリモコン落ちでの暴走を防ぐ。Processing のキープアライブ
再送はこれに合わせている。LINE モードではライン消失時に停止する。

## 制御アーキテクチャ

ライントレース（カメラ）・UDP リモコンともに最終出力は `setMotorPercent()` の
1 経路だけを通す。`loop()` でモードに応じて呼び出し元を切り替えるだけで、
駆動ロジック（ESC パルス変換）は共通。新しい制御ソースを足すときも
`setMotorPercent(left, right)` を呼べばよい。

ライン検出は `lineTrace()` がグレースケール QQVGA フレームの下端帯(ROI)を走査し、
帯平均輝度から作った適応しきい値で線画素の重心を求め、中央からの誤差に P 制御を
かけて左右%を返す。結果は `LineResult` にまとめ `printTelemetry()` がシリアルに出す。
