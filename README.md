# ESP-NOWリモコン ライントレース (XIAO ESP32-S3 Sense)

XIAO ESP32-S3 Sense がカメラでラインを検出して自律走行する。あわせて Mac(Processing) から
ESP-NOW でモーター出力(%)を送るリモコン駆動にも切り替えられる（同じ駆動経路を共有）。
ライン検出の状況はシリアルモニタ(115200)で常時確認できる。

Mac は ESP-NOW を直接話せないため、USB 接続した**中継 ESP32** がシリアル↔ESP-NOW を変換する。
SoftAP/UDP で起きていた「SSID が出ない／接続できない」を避けるためこの方式にした
（旧 UDP 版は `esp32_motor_udp/`・`processing_remote/` にバックアップとして残してある）。

## 構成

```
[Mac / Processing] --USBシリアル("L,R" %)--> [中継ESP32] --ESP-NOW--> [走行ESP32-S3] --ESC信号--> [左右モーター]
   キーボードで%入力                          ブロードキャスト送信        カメラでライン検出
   バーで可視化                                                         フェイルセーフ停止
```

- `esp32_motor_espnow/`       … 走行機(ESP32-S3)のファーム（ライントレース＋ESP-NOW受信）
- `esp32_espnow_bridge/`      … 中継機(予備ESP32)のファーム（シリアル→ESP-NOW送信）
- `processing_remote_serial/` … PC で動かすリモコン（Processing、中継機へシリアル送信）
- `esp32_motor_udp/` / `processing_remote/` … 旧 SoftAP/UDP 版（バックアップ。通常は使わない）
- `モーターテスト/`           … 元のモーター単体テスト（参考）

## セットアップ

### 走行機 ESP32-S3 側

対象ボードは **Seeed Studio XIAO ESP32-S3 Sense**（ESP32-S3, 8MB PSRAM, OV2640）。
カメラピンはこのボード前提でファイル冒頭の `CAM_*` に定義済み。別ボードなら `CAM_*` を差し替える。

1. Arduino IDE のライブラリマネージャで **ESP32Servo** をインストール
   （カメラ用 `esp_camera`・ESP-NOW は ESP32 コアに同梱）
2. `esp32_motor_espnow/esp32_motor_espnow.ino` を開く
3. ファイル先頭の設定を確認
   - `MOTOR_L_PIN=1` (D0) / `MOTOR_R_PIN=2` (D1) … カメラ非使用GPIO。配線に合わせて変更可
   - `LINE_IS_DARK` … 明るい床に黒線=`true` / 暗い床に白線=`false`
   - `BASE_SPEED` / `KP` / `KD` / `MAX_TURN` / `CORNER_SLOWDOWN` / `CORNER_MAX_SPEED` / `CORNER_MIN_SPEED` / `ROI_TOP_R` / `STEER_DEADZONE` / `NORM_FILT_A` … 基本速度・旋回・蛇行制動・旋回上限・カーブ減速・コーナー速度上限/下限・先読み量・直進不感帯・重心平滑化（要現場調整）
   - `ESPNOW_CHANNEL` … 中継機と**必ず同じ値**にする（既定 1）
4. ボードを `XIAO_ESP32S3`、**PSRAM を `OPI PSRAM`** にして書き込み
5. シリアルモニタ(115200)で `カメラ初期化OK` と `ESP-NOW受信待機 ch=1` を確認

### 中継機 ESP32 側

1. 予備の ESP32（走行機とは別の1台）を Mac に USB 接続する
2. `esp32_espnow_bridge/esp32_espnow_bridge.ino` を開く
3. `ESPNOW_CHANNEL` が走行機と同じ（既定 1）であることを確認して書き込み
4. シリアルモニタで `ESP-NOW中継ブリッジ起動 ch=1` が出れば OK。以後 Mac に挿しっぱなしにする

### Mac / Processing 側

1. `processing_remote_serial/processing_remote_serial.pde` を Processing で開く
2. 実行するとコンソールに `Serial.list()` が出る。**中継機のポート名**（Mac は `/dev/cu.usbmodem...`）を
   確認し、`BRIDGE_PORT` に設定して再実行する（空のままなら末尾ポートを自動選択）
3. WiFi 接続は不要（ESP-NOW なので SSID に繋ぐ操作がそもそも無い）

## 動作モード（シリアルで切替）

ESP32 はシリアルモニタに 1 文字送ると動作モードが切り替わる。**起動時は IDLE**。

| 入力 | モード | 動作 |
|------|--------|------|
| `i` | IDLE（既定） | モーターは回さず、ライン検出ログだけ出す。**車輪を回さず検証**できる |
| `l` | LINE | カメラのライン検出で自律走行。ライン消失で停止。**基準速度は Processing から変更可**（下記） |
| `r` | REMOTE | Processing からの ESP-NOW リモコンで駆動 |
| `s` | 停止 | 即停止して IDLE へ |
| `c` | CALIB | 床と線の輝度を実測し、線の明暗・マージンを自動較正（NVS 保存） |
| `x` | 較正消去 | 較正データを消して `LINE_MARGIN` の適応式に戻す |
| `m` | MOTOR | 左右モーターのゲイン・中立を較正（車輪を浮かせて実施、NVS 保存） |

どのモードでも検出処理は毎ループ走るので、ログだけは常に出る。

### 起動時オートスタート（バッテリー単体走行）

モバイルバッテリー等で**PCに繋がず単体起動**したときは、起動後に約 `BOOT_AUTOSTART_SEC`
秒（既定3秒）のカウントダウンを経て**自動で LINE（自律走行）に入る**。スイッチ操作なしで
電源を入れるだけで走り出せる。

```
[BOOT] 3秒後にLINE自律走行を開始します
       中止して検証(IDLE)するなら何かキーを送信...
  3...
  2...
  1...
>> モード: LINE (自律走行) を自動開始
```

- **PC で開発中は、カウントダウン中に何かキーを送れば IDLE のまま留まる**（書き込み直後の暴走防止）。間に合わなければ `s` で停止
- **カメラ初期化に失敗したときは自動走行せず IDLE で待機**（安全策）
- カウントダウン秒数は `BOOT_AUTOSTART_SEC` で変更可
- 自動 LINE 開始後も、ライン消失時は従来どおり停止する

## ライントレースの確認（シリアルモニタ）

シリアルモニタ(115200) に 150ms ごとに検出状況が出る。これを見れば
カメラが線を捉えているか・どう曲がろうとしているかが分かる。

```
[LINE] det=Y mean=128 cx= 92/160 err= +12 px= 340 base= 30% -> L= +41% R= +19% | mode=IDLE (出力のみ)
[LINE] det=N mean=131 px=   3 -- ライン消失 -- | mode=LINE -> 停止
```

| 項目 | 意味 |
|------|------|
| `det` | ライン検出有無（Y/N） |
| `mean` | 注目帯(ROI)の平均輝度。露出が極端なら照明やしきい値を調整 |
| `cx` | 線の重心X(0〜画像幅)。中央が幅の半分 |
| `err` | 中央からのずれ。右に外れると `+`、左で `-` |
| `px` | 線とみなした画素数。`MIN_LINE_PIX` 未満で「消失」 |
| `base` | LINE モードの基準速度%。既定は `BASE_SPEED`、Processing から変更可 |
| `L/R` | P制御で算出した左右出力%。LINE モードのときだけ実際に駆動 |

**検証手順**: 起動（IDLE）→ 車体を手に持ってカメラを線に向ける → `det=Y` と
`cx`/`err` が線の位置に追従して動けば検出 OK → `L=`/`R=` が線を追う向きに変化すれば
制御 OK → 床に置いて `l` を送れば自律走行。追従が弱い/振れるなら `KP` を調整する。
照明や床色でうまく線を拾えないときは、`LINE_IS_DARK`（線の明暗）・`LINE_MARGIN`
（周囲の局所平均からの離れ幅）・`LOCAL_BG_R`（局所平均をとる横幅。広いほど大きな影に強いが、
線幅に近づけすぎると線を見失う）を現場で調整する。

> **較正機能について**: 線検出は較正の有無に関わらず、各画素を周囲の局所平均 ±マージンと
> 比べる**ローカル適応式**（影・外光の明るさムラに強い）で動く。シリアルから `c`（床/線の輝度を
> 実測して線の明暗とマージンを NVS 保存）、`x`（較正データ消去 → `LINE_MARGIN` に戻す）、
> `m`（左右モーターのゲイン・中立を較正、車輪を浮かせて実施）が使える。較正しない場合は
> 適応式のまま走り、`INVERT_L/R`・`DEADBAND_L/R_US`・`SERVO_MID_US` を手で詰めてもよい。

## リモコン操作（REMOTE モード時 / Processing）

| 入力 | 動作 |
|------|------|
| `50` + Enter | 両輪 50% 前進 |
| `50,30` + Enter | 左50% 右30%（差動・カーブ） |
| `-40` + Enter | 両輪 40% 後退 |
| `s` / Space | 即停止 (0%) |
| ↑ / ↓ | 両輪を ±5% 微調整して即送信 |

値の範囲は **-100〜100 (%)**。画面に現在の出力%とバーが表示される。

> **REMOTE 中はカメラを止めてループを軽く保つ**。カメラ取り込みでループが伸びると
> 指令応答が遅れ、フェイルセーフ誤発火でガクつくため。ESP-NOW 受信はコールバックで
> 最新指令を取り込むので、キープアライブや連打でも遅延が溜まらない。

## LINE モードの速度を Processing から変える

`l`（LINE）モード中に Processing から値を送ると、**その値が自律走行の基準速度になる**
（左右独立で来た場合は左右の平均を採用）。操舵（曲がり）はカメラの PD 制御が担うので、
Processing 側は **速度ツマミ**として使える。

| Processing 入力 | LINE モードでの効果 |
|------|------|
| `40` + Enter | 基準速度 40%（直進時 40、旋回時はここから±） |
| `0` / `s` | 基準速度 0 = 実質停止（ラインは検出し続ける） |
| ↑ / ↓ | 基準速度を ±5% 調整 |

シリアルログの `base=` で現在の基準速度を確認できる。値はモードで意味が変わる点に注意：
**REMOTE では左右出力そのもの／LINE では基準速度**。起動時の既定は `BASE_SPEED`(=30)。

## 通信プロトコル

Mac→中継機は USB シリアル、中継機→走行機は ESP-NOW。ペイロードの中身は両区間で同一。

- Mac→中継機: USB シリアル(115200)、**改行(`\n`)終端の 1 行 = 1 指令**
- 中継機→走行機: ESP-NOW ブロードキャスト（宛先 `FF:FF:FF:FF:FF:FF`、ペアリング不要）
- ペイロード: ASCII テキスト（改行は中継機が除いて送る）
  - `"L,R"` … 左右独立（例 `"50,30"`、-100〜100 の整数）
  - `"V"`  … 単一値で両輪同値（例 `"50"`）
  - `"l"` / `"r"` / `"i"` / `"s"` … モード切替コマンド
- 受信側の解釈はモード依存：**REMOTE は左右出力／LINE は基準速度（左右平均）**
- Processing は直近の指令を **200ms間隔で再送**（キープアライブ）
- **チャンネルは送受信で一致必須**（`ESPNOW_CHANNEL`、既定 1）。ズレると一切届かない

## フェイルセーフ

REMOTE モードでは最後の有効指令から **500ms** 指令が途切れると自動で 0% 停止する
（`FAILSAFE_MS`）。通信断やリモコン落ちでの暴走を防ぐ。Processing のキープアライブ
再送はこれに合わせている。LINE モードではライン消失時に停止する。

ESP-NOW はコネクションレスなので、一時的に電波が途切れても次の指令パケットで自動復帰する
（SoftAP のように再接続のハンドシェイクが要らない）。

## 制御アーキテクチャ

ライントレース（カメラ）・ESP-NOW リモコンともに最終出力は `setMotorPercent()` の
1 経路だけを通す。`loop()` でモードに応じて呼び出し元を切り替えるだけで、
駆動ロジック（ESC パルス変換）は共通。新しい制御ソースを足すときも
`setMotorPercent(left, right)` を呼べばよい。ESP-NOW 受信はコールバックでは
バッファに退避するだけにして、`loop()` で `processCommand()` が解釈する。

ライン検出は `lineTrace()` がグレースケール QQVGA フレームの ROI 帯（`ROI_TOP_R`〜`ROI_BOT_R`、
やや先=上を見て高速時のコーナーを早く捉える先読み）を走査し、
各画素を横方向の局所平均と比べるローカル適応しきい値(影・外光に強い)で線画素の重心を求め、中央からの誤差に PD 制御を
かけて左右%を返す（D項 `KD` が誤差の変化に反応し、高速時の蛇行を制動する）。
中央付近の誤差は `STEER_DEADZONE` の不感帯で直進扱いにし、直線での微小ノイズによる
蛇行を抑える（直線では減速しないので速度は犠牲にしない）。
急カーブでは `CORNER_SLOWDOWN` で基準速度を落としつつ、コーナーと判定したら
（`|正規化誤差|` > `CORNER_CAP_AERR`）速度を `CORNER_MAX_SPEED`〜`CORNER_MIN_SPEED` で挟む。
上限は速すぎる脱線を防ぎ（検知時点で前倒し）、下限は減速しすぎの失速・急ピボットによる
ライン消失停止を防ぐ。外輪が飽和した分は内輪から引いて左右差を保つ（高速コーナー対策）。
結果は `LineResult` にまとめ `printTelemetry()` がシリアルに出す。
