# ライントレースの仕組み（高校生向け解説）

このロボットは、**カメラで床の線を見て、左右のタイヤの速さを変えながら線の上を走る**車です。
人が運転しなくても、自分でカーブを曲がりながら線をたどっていきます。

ここでは「どうやって線を見つけて」「どうやってタイヤを動かしているのか」を、
順番に追いかけて説明します。

---

## 全体の流れ（1秒間に何十回もこれを繰り返す）

```
カメラで写真をとる
      ↓
写真の中から「線」を見つける
      ↓
線が画面の右・左・真ん中どこにあるか調べる
      ↓
ズレに合わせて「どれくらい曲がるか」を計算する
      ↓
左右のタイヤの速さを決めてモーターを回す
      ↓
（最初にもどって繰り返す）
```

人間が自転車に乗るとき、「ちょっと右にズレたな → 左にハンドルを切る」
を無意識に何度も繰り返していますよね。あれと同じことを、ロボットは
**1秒間に何十回も**やっています。

---

## ステップ1：カメラで写真をとる

ロボットの頭にカメラ（OV2640）がついています。
カラーではなく**白黒（明るさだけ）の小さな写真**（160×120ピクセル）をとります。

なぜ白黒かというと、ライントレースに必要なのは「色」ではなく
**「明るいか・暗いか」だけ**だからです。
データが軽くなって、処理がすごく速くなります。

### 画面全部は見ない（ROI＝注目エリア）

写真全部を見るのではなく、**画面の下のほう（手前の床）だけ**を見ます。

```
┌─────────────┐
│             │ ← ここは見ない（遠くの景色）
│             │
│░░░░░░░░░░░░░│ ← ここだけ見る（ROI = 足元の線）
└─────────────┘
```

遠くの線を見てもすぐには曲がれないので、**今まさに通過する足元**を見るわけです。
コードでは画面の高さの 65%〜95% の帯（おび）だけを使っています。

---

## ステップ2：写真の中から「線」を見つける

### 「線」とは「まわりより暗い（または明るい）ところ」

白い床に黒い線が引いてあるとします。
カメラから見ると、線の部分だけ**まわりより暗く**写ります。

そこでロボットはこう考えます：

1. まず足元エリア全体の**平均の明るさ**を計算する（＝床の明るさ）
2. そこから**一定以上暗いピクセル**を「これは線だ！」と判定する

このやり方の良いところは、**部屋が明るくても暗くても自動で対応できる**ことです。
「明るさ100より暗ければ線」と固定の数字で決めると、照明が変わったときに
失敗しますが、「**まわりの平均より暗ければ線**」とすれば、明るい部屋でも
暗い部屋でも同じように線を見つけられます。

この「まわりの平均より暗ければ線」という基準は、常に最新の足元の明るさから
作り直すので、**コースや照明が変わっても自動でついていけます**。
（線の明暗や反応の強さを変えたいときは、プログラム先頭の `LINE_IS_DARK` と
`LINE_MARGIN` という数字を調整します。）

---

## ステップ3：線が右・左・真ん中どこにあるか調べる（重心）

線だと判定したピクセルが、画面の左に固まっているのか右に固まっているのか。
それを知るために**「重心（じゅうしん）」**を計算します。

重心とは、ざっくり言うと**「線の真ん中の位置」**です。
線ピクセルのX座標を全部足して個数で割れば、平均の位置＝真ん中が出ます。

```
画面の真ん中
       │
┌──────┼──────┐
│      │  ███  │   線が右にある
└──────┼──────┘
       │   ↑
       │   重心はここ → 「右にズレてる！」
```

そして**「重心」と「画面の真ん中」のズレ**を計算します。これを **誤差（ごさ）** と呼びます。

- 線が右にあれば 誤差はプラス
- 線が左にあれば 誤差はマイナス
- 線が真ん中なら 誤差はゼロ（＝まっすぐでOK）

### 線が見つからなかったら止まる

もし線ピクセルが少なすぎたら（8個未満）、「**線を見失った**」と判断して
**安全のため止まります**。コースアウトして暴走しないための仕組みです。

---

## ステップ4：どれくらい曲がるかを計算する（ここが頭脳）

誤差（線がどれくらいズレているか）がわかったら、
**「じゃあどれだけ曲がればいいか」**を決めます。

### 基本：ズレが大きいほど大きく曲がる（P制御）

いちばん基本の考え方はシンプルで、
**「ズレが大きいほど、強く曲がる」**です。これを **P制御** といいます。

```
曲がる量 ＝ ズレの大きさ × 0.9（KPという調整つまみ）
```

- ちょっとだけ右にズレた → 少しだけ左に曲がる
- 大きく右にズレた → ぐいっと左に曲がる

人間が運転するときと同じで、大きくズレているときほど
ハンドルを大きく切る、という当たり前の感覚です。

### 改良その1：曲がりすぎを抑える（D制御）

P制御だけだと、勢いがついて**曲がりすぎて反対側にズレる**ことがあります。
そして今度は逆に曲がりすぎて…と、左右にフラフラ揺れる**「蛇行（だこう）」**が
起きてしまいます。特にスピードが速いときに起きやすい。

そこで **D制御** を足します。これは
**「ズレが急に縮まっているなら、もう曲がるのをゆるめる」**という働きです。

車庫入れで壁にだんだん近づくとき、ぶつかる手前で早めにブレーキを
ゆるめますよね。あの「先読み」と同じで、**揺れを先回りして抑えます**。

P制御とD制御を合わせて **PD制御** と呼びます。

### 改良その2：直線ではフラフラさせない（不感帯）

まっすぐな直線を走っているとき、カメラのちょっとしたノイズで
重心が左右に1〜2ピクセル動くことがあります。これに反応すると
直線なのにフラフラしてしまう。

そこで**「真ん中のごく近くにいるときは、ズレを無視してまっすぐ走る」**
という**鈍感ゾーン（不感帯）**を作っています。
さらに重心の動きを**ならして（平均化して）**ノイズを消す処理も入れています。

### 改良その3：急カーブでは自動で減速する

スピードが速すぎると、急カーブを曲がりきれずにコースアウトします。
そこで**「ズレが大きい＝急カーブだ」と判断したら、自動でスピードを落とす**
仕組みが入っています。直線では落とさないので、速さは損なわれません。

### 改良その4：車体が斜めなのを見抜いてふらつきを抑える

カーブを抜けた直後、線の真ん中に戻っていても**車体だけ斜めを向いている**ことが
あります。位置（重心）だけ見ていると「真ん中だからOK」と思ってしまい、その後
ズレてから慌てて直す…を繰り返して、直線なのにフラフラします。

そこで足元エリアを**手前（画面の下）と先（画面の上）の2つ**に分け、それぞれで
線の位置を出します。車体がまっすぐなら2つは同じ位置に、**車体が斜めだと線が
斜めに写って2つがズレます**。このズレ＝車体の傾きなので、

```
先の線の位置 － 手前の線の位置 ＝ 車体の傾き
```

を曲がる量に少し足します（`K_ANGLE` という調整つまみ）。こうすると**ズレる前に
傾きを見つけて自分でまっすぐに戻す**ので、カーブ後のふらつきが減ります。
車の運転でも、ハンドルを戻すとき「車体の向き」を見て早めに調整しますよね。
あれと同じ「向きを見て先回りする」工夫です。

---

## ステップ5：左右のタイヤの速さを決めて走る

このロボットには**ハンドルがありません**。
ではどうやって曲がるかというと、**左右のタイヤの速さを変えて曲がります**。

戦車やキャタピラ車と同じ原理です。

```
左タイヤの速さ ＝ 基本スピード ＋ 曲がる量
右タイヤの速さ ＝ 基本スピード － 曲がる量
```

### 具体例：線が右にズレているとき

線が右にあるので、ロボットは右に曲がりたい。すると…

- 左タイヤ：速くする（基本＋曲がる量）
- 右タイヤ：遅くする（基本－曲がる量）

**外側（左）のタイヤを速く、内側（右）のタイヤを遅く**すると、
車は遅いほうの右へ曲がっていきます。
自転車やボートで、片方だけ強く漕ぐと漕いだ反対側に曲がるのと同じです。

```
   左タイヤ 速い ━━━▶
                      ＼ 右に曲がる
   右タイヤ 遅い  ━▶  ／
```

線が真ん中なら、左右同じ速さ＝まっすぐ進みます。

---

## ステップ6：モーターを実際に回す（信号の話）

計算した「速さ（%）」を、モーターが理解できる**信号**に変えて送ります。

このロボットのモーターは**ESC（電子スピードコントローラー）**という部品で
動いていて、ラジコンと同じ仕組みです。信号は
**「パルスの長さ（マイクロ秒）」**で速さを伝えます。

- 1500 マイクロ秒 → **停止**（真ん中）
- それより長い → **前進**
- それより短い → **後進**

つまり「速さ50%」を「1500から少し離れたパルス」に翻訳して送るわけです。

### 現実のモーターに合わせた細かい工夫

教科書どおりにいかないのが現実で、いくつか補正が入っています。

- **動き出しの底上げ（不感帯補正）**：モーターは弱い信号だと
  摩擦で**まったく動きません**。なので「1%でも動かしたい」ときは、
  最低限動き出すところまで信号を底上げします。
- **左右の個体差**：同じ信号を送っても左右のモーターは微妙に速さが
  違うことがあります。まっすぐ走らないときは、プログラム先頭の
  中立値（`SERVO_MID_US`）や動き出しの底上げ量で調整します。
- **回転方向の反転**：タイヤが逆向きについていると前進したいのに
  バックするので、ソフト側で向きを反転できます（`INVERT_L/R`）。

## おまけ：3つの動作モード

このロボットには3つのモードがあります（キーボードで切り替え）。

| キー | モード | 何をする？ |
|------|--------|-----------|
| `l` | LINE | カメラを見て**自分で線をたどって走る**（本命） |
| `r` | REMOTE | 人がスマホ／PCから**ラジコンのように操作** |
| `i` | IDLE | **タイヤは止めたまま**、線を見つけているか画面で確認だけ |

`i`（IDLE）が便利で、**タイヤを回さずに「ちゃんと線を見つけられているか」を
画面の文字で確認**できます。いきなり走らせて暴走させず、まず目だけ
テストできるわけです。

また、PCにつながず**バッテリーだけで電源を入れると、3秒後に自動で
走り出します**（PCをつないでキーを押せば止まったまま）。
本番ではスイッチを入れるだけで走り出せるようになっています。

---

## まとめ：1行で言うと

> **カメラで足元の線を見て、線が右ならちょっと右、左ならちょっと左へ、
> 左右のタイヤの速さを変えながら、1秒に何十回も微調整して走っている。**

これを支えているのが、

1. **白黒画像から線を見つける**（明るさで判定）
2. **線のズレを測る**（重心と誤差）
3. **ズレから曲がる量を計算する**（PD制御＋いろんな改良）
4. **左右タイヤの速さの差で曲がる**（戦車と同じ原理）

という4つの仕組みです。
